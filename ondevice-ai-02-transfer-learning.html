오늘 배울 것

가위바위보 인식 모델이 어느 정도 동작하면 다음 질문이 생긴다.

조명이 바뀌면 왜 틀릴까?
손이 작게 보이면 왜 못 맞힐까?
배경이 복잡하면 왜 헷갈릴까?
성능을 더 올리려면 처음부터 큰 모델을 학습해야 할까?

답은 전이학습이다.

전이학습은 이미 많은 이미지를 보고 배운 모델을 가져와서, 내 문제에 맞게 마지막 부분만 다시 학습하는 방법이다.

하지만 전이학습을 바로 적용하기 전에 먼저 해야 할 일이 있다. 모델이 틀린 이미지를 모으고, 왜 틀렸는지 분류하는 것이다.

오인식 원인 예시 개선 방향
조명 역광, 어두운 교실 밝기/대비 보강 데이터 추가
거리 손이 너무 작음 다양한 거리 데이터 추가
배경 책상 물건과 손이 겹침 복잡한 배경 데이터 추가
포즈 손가락 각도가 애매함 헷갈리는 사례를 더 수집
사람 차이 특정 학생 손만 잘 맞힘 여러 사람 데이터 추가

전이학습은 마법처럼 성능을 올리는 기술이 아니다. 오인식 원인을 데이터에 반영할 때 효과가 커진다.

전이학습을 비유로 이해하기

처음부터 모델을 학습시키는 것은 초등학생에게 사진이 무엇인지부터 가르치는 것과 비슷하다.

전이학습은 이미 사진을 많이 본 학생에게 이렇게 말하는 것과 같다.

"너는 이미 선, 색, 모양, 물체의 윤곽을 알고 있지?
이제 손 모양 중에서 가위, 바위, 보만 구분해 보자."

그래서 데이터가 적어도 성능을 높일 수 있다.

ResNet으로 분류 성능 높이기

ResNet은 이미지 분류에서 많이 쓰이는 CNN 구조다. 핵심 아이디어는 깊은 신경망도 학습이 잘 되도록 shortcut 연결을 넣은 것이다.

쉽게 말하면, 정보가 깊은 층을 지나가다가 사라지지 않도록 우회도로를 만들어 준다.

입력 이미지
→ ResNet feature extractor
→ 새 classification head
→ scissors / rock / paper

여기서 feature extractor는 이미지에서 쓸 만한 특징을 뽑는 부분이다.

초기 층: 선, 모서리, 색 변화
중간 층: 질감, 손가락 일부, 윤곽
깊은 층: 손 모양에 가까운 복합 특징

마지막 classification head는 이 특징을 보고 내 수업의 세 클래스 중 하나로 바꾸는 부분이다.

feature extractor: 이미 배운 눈
classification head: 우리 문제에 맞춘 판단 기준

데이터가 많지 않은 수업 환경에서는 앞부분을 고정하고 마지막 부분만 학습하는 방식이 안정적이다.

ResNet 전이학습 절차

  1. ImageNet으로 학습된 ResNet을 불러온다.
  2. 앞부분 feature extractor는 대부분 유지한다.
  3. 마지막 분류층을 3개 클래스로 바꾼다.
  4. 가위바위보 데이터로 다시 학습한다.
  5. TFLite로 변환해 라즈베리파이에 올린다.

개념 코드는 다음과 같다.

import tensorflow as tf

base = tf.keras.applications.ResNet50(
    include_top=False,
    weights="imagenet",
    input_shape=(224, 224, 3),
)
base.trainable = False

model = tf.keras.Sequential([
    base,
    tf.keras.layers.GlobalAveragePooling2D(),
    tf.keras.layers.Dense(128, activation="relu"),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(3, activation="softmax"),
])

model.compile(
    optimizer="adam",
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

수업에서는 이 코드를 외우게 하기보다 구조를 이해시키는 것이 중요하다.

이미지 특징을 뽑는 부분은 빌려오고
가위바위보를 판단하는 마지막 부분만 새로 붙인다.

처음에는 base.trainable = False로 두고 학습한다. 이렇게 하면 ResNet의 기존 지식은 유지하고 새로 붙인 분류층만 학습한다.

성능이 어느 정도 나온 뒤에는 마지막 몇 개 층만 풀어서 미세 조정할 수 있다.

base.trainable = True
for layer in base.layers[:-20]:
    layer.trainable = False

이것을 fine-tuning이라고 한다. 단, 데이터가 적을 때 너무 많은 층을 학습시키면 기존 지식이 깨지고 과적합될 수 있다.

ResNet 학습 전체 예제

아래 코드는 폴더형 데이터셋을 읽고 ResNet 전이학습을 수행하는 기본 예제다.

import tensorflow as tf

DATA_DIR = "rps_dataset"
IMG_SIZE = (224, 224)
BATCH_SIZE = 16
SEED = 42

train_ds = tf.keras.utils.image_dataset_from_directory(
    f"{DATA_DIR}/train",
    image_size=IMG_SIZE,
    batch_size=BATCH_SIZE,
    seed=SEED,
)

val_ds = tf.keras.utils.image_dataset_from_directory(
    f"{DATA_DIR}/val",
    image_size=IMG_SIZE,
    batch_size=BATCH_SIZE,
    seed=SEED,
)

class_names = train_ds.class_names
print("class names:", class_names)

AUTOTUNE = tf.data.AUTOTUNE
train_ds = train_ds.cache().shuffle(500).prefetch(AUTOTUNE)
val_ds = val_ds.cache().prefetch(AUTOTUNE)

data_augmentation = tf.keras.Sequential([
    tf.keras.layers.RandomFlip("horizontal"),
    tf.keras.layers.RandomRotation(0.08),
    tf.keras.layers.RandomZoom(0.1),
    tf.keras.layers.RandomContrast(0.2),
])

base = tf.keras.applications.ResNet50(
    include_top=False,
    weights="imagenet",
    input_shape=(224, 224, 3),
)
base.trainable = False

inputs = tf.keras.Input(shape=(224, 224, 3))
x = data_augmentation(inputs)
x = tf.keras.applications.resnet50.preprocess_input(x)
x = base(x, training=False)
x = tf.keras.layers.GlobalAveragePooling2D()(x)
x = tf.keras.layers.Dropout(0.3)(x)
outputs = tf.keras.layers.Dense(len(class_names), activation="softmax")(x)

model = tf.keras.Model(inputs, outputs)

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-3),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

history = model.fit(
    train_ds,
    validation_data=val_ds,
    epochs=10,
)

model.save("rps_resnet_saved_model")

이 코드에서 중요한 부분은 세 곳이다.

코드 의미
base.trainable = False ResNet의 기존 지식은 고정
preprocess_input() ResNet이 학습될 때 쓰던 입력 방식으로 변환
Dense(len(class_names)) 내 데이터셋의 클래스 수에 맞게 출력층 생성

학습이 끝난 뒤에는 클래스 순서를 반드시 저장한다.

with open("labels.txt", "w", encoding="utf-8") as f:
    for name in class_names:
        f.write(name + "\n")

TFLite 추론 코드에서 label 순서가 다르면 모델은 맞게 예측했는데 화면에는 다른 이름이 표시될 수 있다.

Fine-tuning 예제

처음 학습 후 검증 정확도가 더 올라갈 여지가 있으면 마지막 일부 층만 풀어 fine-tuning을 한다.

base.trainable = True

for layer in base.layers[:-20]:
    layer.trainable = False

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-5),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

fine_history = model.fit(
    train_ds,
    validation_data=val_ds,
    epochs=5,
)

model.save("rps_resnet_finetuned_saved_model")

fine-tuning에서는 learning rate를 작게 잡는다. 이미 배운 특징을 크게 흔들지 않고, 가위바위보 데이터에 조금만 맞추기 위해서다.

데이터셋 폴더 구조

분류 모델 학습에서는 폴더 구조를 클래스 이름과 맞추면 관리가 쉽다.

rps_dataset/
  train/
    scissors/
    rock/
    paper/
  val/
    scissors/
    rock/
    paper/

학습 데이터와 검증 데이터는 반드시 나눈다.

train: 모델이 공부하는 데이터
val: 모델이 처음 보는 시험 데이터

검증 데이터를 따로 두지 않으면 모델이 외운 것인지 실제로 일반화한 것인지 알 수 없다.

이미지가 한 폴더에 섞여 있다면 다음처럼 간단히 폴더를 나눌 수 있다.

from pathlib import Path
import random
import shutil

SOURCE_DIR = Path("raw_rps")
TARGET_DIR = Path("rps_dataset")
VAL_RATIO = 0.2
random.seed(42)

for class_dir in SOURCE_DIR.iterdir():
    if not class_dir.is_dir():
        continue

    images = list(class_dir.glob("*.jpg")) + list(class_dir.glob("*.png"))
    random.shuffle(images)

    val_count = int(len(images) * VAL_RATIO)
    val_images = set(images[:val_count])

    for image_path in images:
        split = "val" if image_path in val_images else "train"
        out_dir = TARGET_DIR / split / class_dir.name
        out_dir.mkdir(parents=True, exist_ok=True)
        shutil.copy2(image_path, out_dir / image_path.name)

수업에서는 데이터셋을 나눌 때 같은 사진을 train과 val에 동시에 넣으면 안 된다는 점을 강조한다.

현재 RPS 데이터셋을 그대로 학습하기

Desktop에서 확인한 실습 코드는 RPS_Dataset/0, RPS_Dataset/1, RPS_Dataset/2처럼 숫자 폴더를 그대로 라벨로 사용한다. 이 구조는 image_dataset_from_directory()보다 OpenCV와 train_test_split()을 쓰는 방식이 더 직접적이다.

import glob
import cv2
import numpy as np
from sklearn.model_selection import train_test_split

DATASET_DIR = "RPS_Dataset"
IMG_SIZE = 64
LABEL_NAMES = ["scissors", "rock", "paper"]

all_train_x = []
all_train_y = []
all_val_x = []
all_val_y = []

for label_id, label_name in enumerate(LABEL_NAMES):
    files = glob.glob(f"{DATASET_DIR}/{label_id}/*.*")
    print(label_id, label_name, len(files))

    x = []
    y = []

    for path in files:
        image = cv2.imread(path, cv2.IMREAD_COLOR)
        if image is None:
            continue

        image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
        image = cv2.resize(image, (IMG_SIZE, IMG_SIZE))

        x.append(image)
        y.append(label_id)

    x = np.array(x, dtype=np.uint8)
    y = np.array(y, dtype=np.int64)

    x_train, x_val, y_train, y_val = train_test_split(
        x,
        y,
        test_size=0.2,
        random_state=123,
        shuffle=True,
    )

    all_train_x.append(x_train)
    all_train_y.append(y_train)
    all_val_x.append(x_val)
    all_val_y.append(y_val)

train_data = np.concatenate(all_train_x)
train_labels = np.concatenate(all_train_y)
val_data = np.concatenate(all_val_x)
val_labels = np.concatenate(all_val_y)

print(train_data.shape, train_labels.shape)
print(val_data.shape, val_labels.shape)

이 코드의 장점은 학생이 이미지가 어떻게 배열로 바뀌는지 직접 볼 수 있다는 점이다.

이미지 파일 경로
→ cv2.imread()
→ BGR을 RGB로 변환
→ 64 x 64 resize
→ NumPy 배열
→ train/validation 분리

DenseNet 전이학습 코드

기존 실습 노트북은 DenseNet121을 사용한다. ResNet 대신 DenseNet을 써도 전이학습의 핵심 구조는 같다.

import tensorflow as tf

dense_net = tf.keras.applications.DenseNet121(
    weights="imagenet",
    include_top=False,
    input_shape=(IMG_SIZE, IMG_SIZE, 3),
)

inputs = tf.keras.Input(shape=(IMG_SIZE, IMG_SIZE, 3))
x = tf.keras.applications.densenet.preprocess_input(inputs)
x = dense_net(x, training=False)
x = tf.keras.layers.GlobalAveragePooling2D()(x)
x = tf.keras.layers.Dropout(0.2)(x)
outputs = tf.keras.layers.Dense(3, activation="softmax")(x)

model = tf.keras.Model(inputs, outputs)

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=0.0005),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

model.summary()

여기서 preprocess_input()은 DenseNet이 ImageNet으로 학습될 때 사용한 입력 스케일에 맞춰 이미지를 바꿔 준다. 그래서 이 코드에서는 train_data = train_data / 255.0을 하지 않는다.

학습 코드는 다음과 같다.

checkpoint = tf.keras.callbacks.ModelCheckpoint(
    "RPS_PreTrained_DenseNet.keras",
    save_best_only=True,
    monitor="val_accuracy",
    mode="max",
)

history = model.fit(
    train_data,
    train_labels,
    epochs=30,
    validation_data=(val_data, val_labels),
    callbacks=[checkpoint],
)

가장 좋은 모델을 다시 불러와 평가한다.

best_model = tf.keras.models.load_model("RPS_PreTrained_DenseNet.keras")

loss, accuracy = best_model.evaluate(val_data, val_labels, verbose=0)
print("validation loss:", loss)
print("validation accuracy:", accuracy)

예측 결과를 눈으로 확인하는 코드도 넣어 두면 좋다.

import matplotlib.pyplot as plt

pred = best_model.predict(val_data)
pred_labels = np.argmax(pred, axis=1)

plt.figure(figsize=(14, 7))
for i in range(10):
    plt.subplot(2, 5, i + 1)
    plt.imshow(val_data[i])
    true_name = LABEL_NAMES[int(val_labels[i])]
    pred_name = LABEL_NAMES[int(pred_labels[i])]
    plt.title(f"true: {true_name}\npred: {pred_name}")
    plt.axis("off")

plt.tight_layout()
plt.show()

수업에서는 정확도 숫자만 보지 말고, 틀린 이미지를 직접 확인하게 해야 한다. 어떤 각도와 배경에서 틀리는지를 봐야 다음 데이터 수집 방향이 나온다.

ImageDataGenerator로 데이터 보강하기

기존 augmentation 실습 코드는 ImageDataGenerator를 사용한다.

img_gen_train = tf.keras.preprocessing.image.ImageDataGenerator(
    horizontal_flip=True,
    rotation_range=0.2,
    width_shift_range=0.1,
    height_shift_range=0.1,
    brightness_range=(0.1, 0.9),
    zoom_range=0.2,
)

train_gen = img_gen_train.flow(
    train_data,
    train_labels,
    batch_size=32,
    seed=1,
)

보강된 이미지를 눈으로 확인하는 코드는 다음과 같다.

batch_x, batch_y = next(train_gen)

plt.figure(figsize=(12, 6))
for i in range(12):
    plt.subplot(3, 4, i + 1)
    plt.imshow(batch_x[i].astype("uint8"))
    plt.title(LABEL_NAMES[int(batch_y[i])])
    plt.axis("off")

plt.tight_layout()
plt.show()

augmentation을 적용한 학습은 다음처럼 진행한다.

checkpoint_aug = tf.keras.callbacks.ModelCheckpoint(
    "RPS_PreTrained_DenseNet_Augmentation.keras",
    save_best_only=True,
    monitor="val_accuracy",
    mode="max",
)

history_aug = model.fit(
    train_gen,
    epochs=100,
    validation_data=(val_data, val_labels),
    callbacks=[checkpoint_aug],
)

여기서 train_gen은 매 epoch마다 조금씩 다른 이미지를 만들어 낸다. 원본 데이터 개수는 그대로지만, 모델은 회전, 이동, 밝기 변화가 들어간 다양한 손 이미지를 보게 된다.

학습한 모델을 TFLite로 저장하기

라즈베리파이에서 쓰려면 Keras 모델을 TFLite로 변환한다.

best_model = tf.keras.models.load_model("RPS_PreTrained_DenseNet_Augmentation.keras")

converter = tf.lite.TFLiteConverter.from_keras_model(best_model)
tflite_model = converter.convert()

with open("RPS_PreTrained_DenseNet_Augmentation.tflite", "wb") as f:
    f.write(tflite_model)

변환 후에는 TFLite 모델을 다시 열어서 입력 shape을 확인한다.

interpreter = tf.lite.Interpreter(
    model_path="RPS_PreTrained_DenseNet_Augmentation.tflite"
)
interpreter.allocate_tensors()

print(interpreter.get_input_details()[0]["shape"])
print(interpreter.get_input_details()[0]["dtype"])
print(interpreter.get_output_details()[0]["shape"])

이 정보가 01회차의 실시간 추론 코드에서 사용하는 IMG_SIZE, input_dtype, label 순서와 맞아야 한다.

데이터 보강이 중요한 이유

라즈베리파이 카메라 환경은 매번 다르다.

밝은 조명
어두운 조명
손이 가까움
손이 멂
배경이 단순함
배경이 복잡함
손이 조금 기울어짐

그래서 학습 때 일부러 이미지를 조금씩 바꿔 보여준다. 이것을 data augmentation이라고 한다.

data_augmentation = tf.keras.Sequential([
    tf.keras.layers.RandomFlip("horizontal"),
    tf.keras.layers.RandomRotation(0.08),
    tf.keras.layers.RandomZoom(0.1),
    tf.keras.layers.RandomContrast(0.2),
])

데이터 보강의 목적은 모델에게 이렇게 가르치는 것이다.

손이 조금 돌아가거나 조명이 바뀌어도 같은 가위바위보로 봐야 한다.

다만 보강을 무조건 많이 넣는 것이 좋은 것은 아니다. 실제 상황에서 일어나지 않는 변형은 오히려 학습을 방해한다.

보강 가위바위보에 적합한가? 이유
좌우 반전 대체로 적합 왼손/오른손 차이에 대응
약한 회전 적합 손이 조금 기울 수 있음
약한 줌 적합 카메라와 거리 변화
강한 색 왜곡 주의 실제 카메라 색과 너무 달라질 수 있음
상하 반전 부적합 뒤집힌 손은 일반 상황이 아님

수업에서는 “현실에서 생길 만한 변화만 넣는다”는 기준을 강조하면 된다.

ResNet과 Tiny YOLO의 차이

ResNet은 분류 모델이다.

이미지 전체를 보고
"이 이미지는 rock입니다"

Tiny YOLO는 객체탐지 모델이다.

이미지 안에서 손 위치를 찾고
"여기 있는 손은 rock입니다"

차이는 매우 중요하다.

모델 답하는 질문 출력
ResNet 이 이미지가 무엇인가? class label
Tiny YOLO 어디에 무엇이 있는가? box + class

학생들이 헷갈리지 않도록 다음 예시를 들면 좋다.

카메라 화면에 손 하나가 크게 보이고 배경이 단순하다.
→ ResNet 같은 분류 모델로 충분할 가능성이 높다.

카메라 화면에 손이 작게 보이고 주변 물체가 많다.
→ 먼저 손 위치를 찾아야 하므로 Tiny YOLO 같은 탐지 모델이 유리하다.

즉, 모델 선택은 더 유명한 모델을 고르는 문제가 아니라 입력 화면에서 답을 내기 위해 어떤 정보가 필요한가를 판단하는 문제다.

언제 Tiny YOLO가 필요한가?

다음 상황에서는 분류 모델보다 객체탐지 모델이 유리하다.

손이 화면 일부에만 작게 보일 때
배경에 다른 물체가 많을 때
여러 사람이 동시에 손을 낼 때
손 위치까지 표시하고 싶을 때

Tiny YOLO는 다음과 같은 결과를 낸다.

{
  "class": "paper",
  "confidence": 0.88,
  "box": [105, 42, 210, 180]
}

여기서 box는 손이 있는 위치다.

Tiny YOLO 전이학습 절차

  1. 손 이미지에 bounding box를 표시한다.
  2. 각 box에 scissors, rock, paper 라벨을 붙인다.
  3. Tiny YOLO 사전학습 모델을 불러온다.
  4. 마지막 탐지 head를 내 클래스 수에 맞게 바꾼다.
  5. 학습 후 TFLite 또는 ONNX 계열로 변환한다.
  6. 라즈베리파이에서 속도를 측정한다.

객체탐지는 분류보다 데이터 준비가 어렵다. 하지만 실제 환경에서는 더 강하다.

객체탐지 데이터에서는 라벨 하나만 붙이는 것이 아니라 손의 위치까지 표시해야 한다.

분류 데이터: 이 사진은 paper다.
탐지 데이터: 이 사진의 (x1, y1, x2, y2) 영역에 paper 손이 있다.

bounding box 품질이 나쁘면 좋은 모델을 써도 성능이 잘 나오지 않는다. 손보다 박스를 너무 크게 잡으면 배경까지 같이 학습하고, 너무 작게 잡으면 손가락 일부가 빠진다.

수업용으로는 처음부터 Tiny YOLO를 학습시키기보다, ResNet 분류 실습 후 “왜 탐지가 필요한가”를 이해시키는 확장 주제로 다루는 편이 좋다.

탐지 결과를 사용하는 코드는 보통 다음 형태가 된다.

detections = [
    {"class": "paper", "confidence": 0.88, "box": [105, 42, 210, 180]},
    {"class": "rock", "confidence": 0.61, "box": [20, 60, 90, 150]},
]

best = None
for det in detections:
    if det["confidence"] < 0.75:
        continue
    if best is None or det["confidence"] > best["confidence"]:
        best = det

if best is not None:
    label = best["class"]
    x1, y1, x2, y2 = best["box"]
    print(label, x1, y1, x2, y2)
else:
    print("no reliable hand detected")

분류 모델은 이미지 전체에서 하나의 답을 고르지만, 탐지 모델은 여러 후보 중 confidence가 높은 박스를 골라야 한다.

TFLite 변환 예제

학습한 SavedModel을 라즈베리파이에서 쓰려면 TFLite로 변환한다.

import tensorflow as tf

converter = tf.lite.TFLiteConverter.from_saved_model("rps_resnet_finetuned_saved_model")
converter.optimizations = [tf.lite.Optimize.DEFAULT]

tflite_model = converter.convert()

with open("rps_resnet_dynamic.tflite", "wb") as f:
    f.write(tflite_model)

변환 후에는 파일 크기를 확인한다.

from pathlib import Path

size_mb = Path("rps_resnet_dynamic.tflite").stat().st_size / 1024 / 1024
print(f"TFLite model size: {size_mb:.2f} MB")

라즈베리파이에 올릴 모델은 정확도뿐 아니라 파일 크기와 추론 속도까지 같이 봐야 한다.

성능을 높이는 체크리스트

문제 해결 방법
조명 바뀌면 틀림 밝기/대비 augmentation 추가
손이 작으면 틀림 다양한 거리의 이미지 추가
배경에 약함 다양한 배경 데이터 추가
특정 사람 손만 잘 맞힘 여러 사람 손 데이터 추가
가위와 보를 헷갈림 헷갈리는 클래스 데이터 추가
실시간 속도 느림 입력 크기 축소, 경량 모델 사용

개선 실험은 한 번에 하나씩 바꿔야 한다.

나쁜 실험: 모델도 바꾸고, 데이터도 바꾸고, 입력 크기도 바꾼다.
좋은 실험: 데이터 보강만 추가하고 결과를 비교한다.

그래야 성능이 좋아졌을 때 무엇 때문에 좋아졌는지 설명할 수 있다.

좋은 데이터셋의 기준

좋은 데이터셋은 예쁜 사진 모음이 아니다. 실제 사용할 환경을 잘 대표해야 한다.

라즈베리파이 카메라로 찍은 데이터
교실 조명에서 찍은 데이터
학생 여러 명의 손
다양한 배경
틀리기 쉬운 각도

모델 성능은 모델 구조보다 데이터 품질에 더 크게 좌우되는 경우가 많다.

데이터를 모을 때는 최소한 다음 정보를 같이 기록하면 좋다.

기록 항목 이유
촬영 장소 배경과 조명 차이 확인
촬영 사람 특정 사람에게만 맞는지 확인
손동작 라벨 클래스 균형 확인
오인식 여부 개선 데이터 선별
confidence 애매한 사례 분석

특히 scissors, rock, paper 데이터 수가 크게 다르면 모델이 많은 클래스 쪽으로 치우칠 수 있다. 세 클래스의 이미지 수를 비슷하게 맞추는 것이 기본이다.

실습 과제

  1. 기존 TFLite 모델이 틀리는 이미지를 30장 모은다.
  2. 왜 틀렸는지 원인을 적는다.
  3. ResNet 전이학습용 데이터셋을 만든다.
  4. augmentation 전후 정확도를 비교한다.
  5. 손 위치까지 찾고 싶다면 Tiny YOLO용 bounding box 데이터를 만들어 본다.

보고서에는 다음 항목을 포함한다.

  1. 기존 모델이 틀린 대표 이미지 5장과 원인
  2. 추가로 수집한 데이터의 클래스별 개수
  3. augmentation 적용 전후의 검증 정확도
  4. 라즈베리파이에서 실행 가능한지에 대한 판단
  5. 다음 개선 방향

오늘의 핵심 정리

전이학습은 이미 배운 모델을 내 문제에 맞게 다시 사용하는 방법이다.
ResNet은 이미지 전체를 분류하는 데 적합하다.
Tiny YOLO는 이미지 안에서 손 위치와 손 모양을 함께 찾는 데 적합하다.
성능 개선의 출발점은 모델 변경보다 오인식 데이터 분석이다.
