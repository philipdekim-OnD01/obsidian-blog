오늘 배울 것

가위바위보 인식 모델이 어느 정도 동작하면 다음 질문이 생긴다.

조명이 바뀌면 왜 틀릴까?
손이 작게 보이면 왜 못 맞힐까?
배경이 복잡하면 왜 헷갈릴까?
성능을 더 올리려면 처음부터 큰 모델을 학습해야 할까?

답은 전이학습이다.

전이학습은 이미 많은 이미지를 보고 배운 모델을 가져와서, 내 문제에 맞게 마지막 부분만 다시 학습하는 방법이다.

전이학습을 비유로 이해하기

처음부터 모델을 학습시키는 것은 초등학생에게 사진이 무엇인지부터 가르치는 것과 비슷하다.

전이학습은 이미 사진을 많이 본 학생에게 이렇게 말하는 것과 같다.

"너는 이미 선, 색, 모양, 물체의 윤곽을 알고 있지?
이제 손 모양 중에서 가위, 바위, 보만 구분해 보자."

그래서 데이터가 적어도 성능을 높일 수 있다.

ResNet으로 분류 성능 높이기

ResNet은 이미지 분류에서 많이 쓰이는 CNN 구조다. 핵심 아이디어는 깊은 신경망도 학습이 잘 되도록 shortcut 연결을 넣은 것이다.

쉽게 말하면, 정보가 깊은 층을 지나가다가 사라지지 않도록 우회도로를 만들어 준다.

입력 이미지
→ ResNet feature extractor
→ 새 classification head
→ scissors / rock / paper

ResNet 전이학습 절차

  1. ImageNet으로 학습된 ResNet을 불러온다.
  2. 앞부분 feature extractor는 대부분 유지한다.
  3. 마지막 분류층을 3개 클래스로 바꾼다.
  4. 가위바위보 데이터로 다시 학습한다.
  5. TFLite로 변환해 라즈베리파이에 올린다.

개념 코드는 다음과 같다.

import tensorflow as tf

base = tf.keras.applications.ResNet50(
    include_top=False,
    weights="imagenet",
    input_shape=(224, 224, 3),
)
base.trainable = False

model = tf.keras.Sequential([
    base,
    tf.keras.layers.GlobalAveragePooling2D(),
    tf.keras.layers.Dense(128, activation="relu"),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(3, activation="softmax"),
])

model.compile(
    optimizer="adam",
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

수업에서는 이 코드를 외우게 하기보다 구조를 이해시키는 것이 중요하다.

이미지 특징을 뽑는 부분은 빌려오고
가위바위보를 판단하는 마지막 부분만 새로 붙인다.

데이터 보강이 중요한 이유

라즈베리파이 카메라 환경은 매번 다르다.

밝은 조명
어두운 조명
손이 가까움
손이 멂
배경이 단순함
배경이 복잡함
손이 조금 기울어짐

그래서 학습 때 일부러 이미지를 조금씩 바꿔 보여준다. 이것을 data augmentation이라고 한다.

data_augmentation = tf.keras.Sequential([
    tf.keras.layers.RandomFlip("horizontal"),
    tf.keras.layers.RandomRotation(0.08),
    tf.keras.layers.RandomZoom(0.1),
    tf.keras.layers.RandomContrast(0.2),
])

데이터 보강의 목적은 모델에게 이렇게 가르치는 것이다.

손이 조금 돌아가거나 조명이 바뀌어도 같은 가위바위보로 봐야 한다.

ResNet과 Tiny YOLO의 차이

ResNet은 분류 모델이다.

이미지 전체를 보고
"이 이미지는 rock입니다"

Tiny YOLO는 객체탐지 모델이다.

이미지 안에서 손 위치를 찾고
"여기 있는 손은 rock입니다"

차이는 매우 중요하다.

모델 답하는 질문 출력
ResNet 이 이미지가 무엇인가? class label
Tiny YOLO 어디에 무엇이 있는가? box + class

언제 Tiny YOLO가 필요한가?

다음 상황에서는 분류 모델보다 객체탐지 모델이 유리하다.

손이 화면 일부에만 작게 보일 때
배경에 다른 물체가 많을 때
여러 사람이 동시에 손을 낼 때
손 위치까지 표시하고 싶을 때

Tiny YOLO는 다음과 같은 결과를 낸다.

{
  "class": "paper",
  "confidence": 0.88,
  "box": [105, 42, 210, 180]
}

여기서 box는 손이 있는 위치다.

Tiny YOLO 전이학습 절차

  1. 손 이미지에 bounding box를 표시한다.
  2. 각 box에 scissors, rock, paper 라벨을 붙인다.
  3. Tiny YOLO 사전학습 모델을 불러온다.
  4. 마지막 탐지 head를 내 클래스 수에 맞게 바꾼다.
  5. 학습 후 TFLite 또는 ONNX 계열로 변환한다.
  6. 라즈베리파이에서 속도를 측정한다.

객체탐지는 분류보다 데이터 준비가 어렵다. 하지만 실제 환경에서는 더 강하다.

성능을 높이는 체크리스트

문제 해결 방법
조명 바뀌면 틀림 밝기/대비 augmentation 추가
손이 작으면 틀림 다양한 거리의 이미지 추가
배경에 약함 다양한 배경 데이터 추가
특정 사람 손만 잘 맞힘 여러 사람 손 데이터 추가
가위와 보를 헷갈림 헷갈리는 클래스 데이터 추가
실시간 속도 느림 입력 크기 축소, 경량 모델 사용

좋은 데이터셋의 기준

좋은 데이터셋은 예쁜 사진 모음이 아니다. 실제 사용할 환경을 잘 대표해야 한다.

라즈베리파이 카메라로 찍은 데이터
교실 조명에서 찍은 데이터
학생 여러 명의 손
다양한 배경
틀리기 쉬운 각도

모델 성능은 모델 구조보다 데이터 품질에 더 크게 좌우되는 경우가 많다.

실습 과제

  1. 기존 TFLite 모델이 틀리는 이미지를 30장 모은다.
  2. 왜 틀렸는지 원인을 적는다.
  3. ResNet 전이학습용 데이터셋을 만든다.
  4. augmentation 전후 정확도를 비교한다.
  5. 손 위치까지 찾고 싶다면 Tiny YOLO용 bounding box 데이터를 만들어 본다.

오늘의 핵심 정리

전이학습은 이미 배운 모델을 내 문제에 맞게 다시 사용하는 방법이다.
ResNet은 이미지 전체를 분류하는 데 적합하다.
Tiny YOLO는 이미지 안에서 손 위치와 손 모양을 함께 찾는 데 적합하다.
성능 개선의 출발점은 모델 변경보다 오인식 데이터 분석이다.
