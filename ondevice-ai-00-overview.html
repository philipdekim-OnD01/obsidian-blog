Ondevice AI 강좌 00: 라즈베리파이로 배우는 전체 지도

시리즈: 00 전체지도 · 01 TFLite 가위바위보 · 02 전이학습 · 03 경량화

이 강좌의 목표

이 강좌는 라즈베리파이 5와 카메라를 이용해 AI 모델이 실제 장치에서 어떻게 동작하는지 배우는 실습형 강좌다.

중심 예제는 가위바위보 인식이다.

카메라 입력
→ 이미지 전처리
→ TFLite 모델 추론
→ 가위/바위/보 판정
→ 게임 로직 또는 외부 장치와 연결

학생이 꼭 가져가야 할 메시지는 하나다.

AI 모델은 노트북 안에서 정확도만 찍는 것이 아니라, 카메라, 센서, 출력 장치, 네트워크와 연결될 때 실제 시스템이 된다.

수업을 시작하기 전에 잡아야 할 관점

온디바이스 AI 수업에서 학생들이 가장 자주 오해하는 부분은 “모델만 있으면 된다”는 생각이다. 실제 장치에서는 모델보다 주변 조건이 더 큰 문제를 만든다.

카메라가 어두우면 입력이 흔들린다.
입력 크기가 맞지 않으면 모델이 실행되지 않는다.
추론이 1초 이상 걸리면 실시간 게임이 되지 않는다.
confidence가 높아도 같은 손동작이 계속 유지되는지 확인해야 한다.

그래서 이 강좌는 모델 구조를 깊게 파는 수업이 아니라, AI 모델을 작은 컴퓨터 안에서 안정적으로 작동시키는 전체 흐름을 배우는 수업이다.

수업 중에는 다음 질문을 계속 던지면 좋다.

질문 확인하려는 것
입력은 모델이 기대하는 형태인가? 이미지 크기, 색상 채널, dtype
출력은 바로 믿어도 되는가? confidence, 연속 프레임 안정성
장치에서 충분히 빠른가? 평균 추론 시간, FPS
틀린 경우를 기록했는가? 조명, 거리, 배경, 손 모양

강좌 구성

회차 제목 핵심 질문
01 TFLite로 실시간 가위바위보 인식하기 작은 모델을 라즈베리파이에서 어떻게 실행할까?
02 전이학습으로 성능 높이기: ResNet과 Tiny YOLO 데이터가 적을 때 성능을 어떻게 올릴까?

학생은 먼저 블로그 설명을 읽고, GitHub 저장소에서 노트북과 데이터를 내려받아 실습하면 된다.

각 회차는 독립된 지식이 아니라 하나의 개선 과정으로 이어진다.

01회차: 일단 돌아가는 시스템을 만든다.
02회차: 틀리는 이유를 분석하고 데이터와 모델을 개선한다.
03회차: 개선한 모델을 실제 장치에서 빠르게 돌릴 수 있게 줄인다.

즉, 좋은 수업 흐름은 “정확한 모델 만들기”가 아니라 작동하는 모델을 만들고, 실패를 관찰하고, 장치 제약 안에서 개선하기다.

먼저 알아둘 용어

용어 쉬운 뜻
On-device AI 서버가 아니라 라즈베리파이, 스마트폰, MCU 같은 장치 안에서 직접 AI를 실행하는 방식
CNN 이미지를 잘 인식하는 신경망 구조
TFLite TensorFlow 모델을 모바일/임베디드 장치에서 실행하기 위한 가벼운 형식
Inference 학습이 끝난 모델에 새 이미지를 넣고 답을 얻는 과정
Transfer Learning 이미 학습된 모델을 가져와 내 문제에 맞게 조금만 다시 학습하는 방법
Quantization 32비트 실수 모델을 8비트 정수처럼 더 작은 숫자 표현으로 바꾸는 경량화 방법
Pruning 중요하지 않은 연결이나 가중치를 제거해 모델을 가볍게 만드는 방법

가위바위보 TFLite 모델

라즈베리파이 5 2GB에서도 작은 CNN/TFLite 모델은 충분히 실습 가능하다. 다만 VLM 같은 큰 멀티모달 모델은 메모리와 속도 때문에 직접 실행하기 어렵다.

처음 실습을 시작할 때는 다음 네 가지를 먼저 확인한다.

확인 항목 확인 방법 문제가 생기면
카메라 인식 OpenCV로 한 프레임 읽기 카메라 번호 0, 1을 바꿔 본다
모델 파일 위치 Python에서 파일 경로 확인 코드와 같은 폴더에 두거나 절대경로를 쓴다
패키지 import import cv2, import numpy 실행 가상환경과 설치 위치를 확인한다
화면 출력 cv2.imshow() 실행 SSH 환경이면 모니터 연결 또는 웹 출력으로 바꾼다

특히 라즈베리파이에서 원격 접속으로 수업할 때는 cv2.imshow() 창이 안 뜨는 경우가 있다. 이때는 모델이 틀린 것이 아니라 화면 출력 환경 문제일 수 있다.

왜 가위바위보인가?

가위바위보는 교육용 예제로 매우 좋다.

  1. 클래스가 3개라 이해하기 쉽다.
  2. 카메라 입력을 바로 사용할 수 있다.
  3. 결과가 즉시 보이기 때문에 학생이 흥미를 느낀다.
  4. 분류, 전이학습, 객체탐지, 경량화까지 확장하기 쉽다.

가위바위보 예제는 단순해 보이지만 실제 AI 시스템의 핵심 문제를 모두 포함한다.

실제 문제 가위바위보에서의 모습
입력 품질 조명, 카메라 흔들림, 손과 카메라 거리
클래스 구분 가위와 보가 비슷하게 보이는 순간
실시간성 화면이 끊기지 않는 FPS
사용자 경험 판정이 너무 빨라도, 너무 늦어도 불편함
시스템 연결 결과를 게임, LED, 웹 화면으로 보내기

따라서 이 예제는 장난감 프로젝트가 아니라 작은 형태의 실제 엣지 AI 프로젝트라고 볼 수 있다.

수업 운영 팁

처음부터 정확도를 너무 강조하지 않는다. 첫 목표는 한 장치 안에서 입력, 모델, 출력이 연결되는 경험이다.

좋은 수업 순서는 다음과 같다.

  1. 카메라가 켜지는지 확인한다.
  2. 모델이 한 장 이미지를 분류하는지 확인한다.
  3. 실시간 영상에 결과를 표시한다.
  4. 오인식 사례를 모은다.
  5. 전이학습과 데이터 보강으로 개선한다.
  6. Quantization과 Pruning으로 속도와 크기를 줄인다.

수업 중 디버깅 순서는 반드시 단순한 것부터 확인한다.

1. 카메라가 켜졌는가?
2. 프레임의 shape이 예상과 같은가?
3. resize 후 입력 shape이 모델 입력과 같은가?
4. 모델 출력 배열 길이가 3인가?
5. label 순서가 모델 학습 순서와 같은가?
6. confidence 기준이 너무 높거나 낮지 않은가?

초보자는 모델이 틀렸다고 바로 판단하지만, 실제로는 label 순서가 바뀌었거나 입력 dtype이 맞지 않는 경우가 많다. 수업에서는 이 과정을 보여 주는 것이 중요하다.

평가 기준

최종 과제는 정확도 하나로만 평가하지 않는다. 온디바이스 AI에서는 다음 항목을 함께 본다.

평가 항목 좋은 결과의 기준
동작 완성도 카메라 입력부터 결과 출력까지 끊기지 않는다
안정성 같은 손동작을 여러 프레임 확인한 뒤 판정한다
설명 가능성 틀린 사례의 원인을 말할 수 있다
장치 적합성 라즈베리파이에서 실시간에 가깝게 동작한다
확장성 JSON, GPIO, 웹 화면 중 하나와 연결할 수 있다

강좌의 마지막 목표

최종 결과물은 단순한 분류 코드가 아니다.

라즈베리파이가 카메라로 손 모양을 보고
가위/바위/보를 판단하고
게임 결과를 JSON으로 내보내거나
LED, 스피커, 웹 화면과 연결하는 작은 AI 시스템

이 흐름을 이해하면 음식 분류, 불량 검출, 안전모 착용 여부 확인, 간단한 제스처 인터페이스 같은 다른 온디바이스 AI 프로젝트로 확장할 수 있다.
