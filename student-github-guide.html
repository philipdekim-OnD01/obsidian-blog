학생을 위한 GitHub 활용 가이드

GitHub는 단순히 코드를 보관하는 웹사이트가 아니다. 개인 노트북, 학교 PC, Google Colab, GCP, Kaggle 같은 외부 연산 환경을 연결하는 출발점이다. 특히 AI 실습에서는 GPU가 있는 클라우드 환경으로 코드를 옮기고, 결과를 공유하고, 프로젝트를 포트폴리오로 정리할 때 유용하다.

한눈에 보는 핵심 장점 10가지

번호 장점 학생에게 유용한 이유
1 클라우드 GPU와 즉시 연결 개인 PC에 GPU가 없어도 Colab, GCP, Kaggle에서 코드를 바로 가져와 학습할 수 있다.
2 여러 기기에서 같은 프로젝트 사용 학교 PC, 개인 노트북, 클라우드 서버를 바꿔도 같은 코드를 사용할 수 있다.
3 최신 실습 자료 다운로드 교수자가 업데이트한 노트북과 데이터를 동일한 링크에서 받을 수 있다.
4 팀 프로젝트 협업 팀원이 각자 작업한 코드를 공유하고 병합할 수 있다.
5 포트폴리오 구축 프로젝트 코드, README, 실행 방법을 URL 하나로 보여 줄 수 있다.
6 오픈소스 실습 공개 AI 프로젝트를 fork 또는 clone하여 분석하고 개선할 수 있다.
7 웹 페이지와 결과물 공개 GitHub Pages를 사용하면 프로젝트 소개 페이지를 무료로 공개할 수 있다.
8 질문과 피드백 공유 오류가 난 파일의 링크를 전달하면 문제 상황을 정확하게 설명할 수 있다.
9 프로젝트 기여 내용 증명 commit과 Pull Request 기록으로 자신이 맡은 작업을 구체적으로 보여 줄 수 있다.
10 실무 방식 연습 clone, branch, commit, push, Pull Request 흐름을 미리 익힐 수 있다.

가장 추천하는 AI 실습 흐름

GitHub에서 실습 코드 받기
→ Colab 또는 GCP GPU 환경에서 clone
→ 모델 학습과 실험
→ 필요한 코드와 문서만 commit
→ 자신의 GitHub 저장소에 push
→ 프로젝트 링크 공유

학습 데이터와 대용량 모델 파일은 GitHub에 무조건 올리지 않는다. 큰 파일은 Google Drive, Hugging Face Hub, Git LFS 같은 별도 저장 방식을 검토한다.

1. GitHub 가입

  1. GitHub 가입 페이지에 접속한다.
  2. 이메일 주소, 비밀번호, 사용자 이름을 입력한다.
  3. 가입 후 이메일 인증을 완료한다.
  4. 로그인한 뒤 오른쪽 위 프로필 메뉴에서 자신의 사용자 이름을 확인한다.

GitHub 공식 안내에 따르면 무료 개인 계정으로 시작할 수 있으며, Google 또는 Apple 계정으로도 가입할 수 있다.

  • 공식 문서: Creating an account on GitHub

사용자 이름은 저장소 주소에 포함된다.

https://github.com/사용자이름/저장소이름

2. Git 설치 확인

macOS 또는 Linux 터미널, Windows PowerShell이나 Git Bash에서 다음 명령을 실행한다.

git --version

버전 번호가 나오면 설치되어 있다. 설치되어 있지 않다면 Git 공식 다운로드 페이지에서 설치한다.

처음 한 번은 자신의 이름과 이메일을 설정한다.

git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

3. 수업 코드 다운로드

이 수업의 전체 코드는 다음 저장소에서 받을 수 있다.

  • RP2.CNN02 GitHub 저장소

터미널에서 다음 명령을 실행한다.

git clone https://github.com/philipdekim-OnD01/RP2.CNN02.git
cd RP2.CNN02

GitHub 공식 문서에서 안내하는 기본 clone 방식이다.

  • 공식 문서: Cloning a repository

Git 명령이 익숙하지 않다면 저장소 페이지의 Code 버튼을 누르고 Download ZIP을 선택해도 된다.

4. Colab GPU에서 수업 코드 사용

Google Colab에서 새 노트북을 열고 다음 셀을 실행한다.

!git clone https://github.com/philipdekim-OnD01/RP2.CNN02.git
%cd RP2.CNN02

Colab 메뉴에서 GPU를 선택한다.

런타임

5. 최신 수업 코드 반영

이미 저장소를 내려받았다면 다시 clone하지 않고 다음 명령을 실행한다.

cd RP2.CNN02
git pull

수업 시작 전에 git pull을 실행하면 교수자가 올린 최신 노트북과 자료를 받을 수 있다.

6. GitHub에서 특정 파일만 다운로드

특정 노트북만 필요하다면 GitHub에서 파일을 연 뒤 Download raw file 버튼을 누른다.

GitHub의 Jupyter Notebook 미리보기가 일시적으로 실패하더라도 raw 파일 다운로드에는 문제가 없는 경우가 많다.

주요 실습 파일:

  • 기본 RPS 분류 노트북
  • 데이터 보강 RPS 노트북
  • 가위바위보 이미지 데이터셋

7. 자신의 저장소 만들기

  1. GitHub에 로그인한다.
  2. 오른쪽 위 + 버튼을 누른다.
  3. New repository를 선택한다.
  4. 저장소 이름을 입력한다.
  5. 공개 포트폴리오라면 Public, 개인 과제라면 Private를 선택한다.
  6. Create repository를 누른다.
  • 공식 문서: Creating a new repository

8. 자신의 코드를 GitHub에 올리기

프로젝트 폴더에서 다음 흐름을 사용한다.

git add .
git commit -m "Add model training notebook"
git push

처음 연결하는 개인 저장소라면 GitHub의 새 저장소 페이지에 표시되는 안내를 따라 remote를 추가한다.

git remote add origin https://github.com/사용자이름/저장소이름.git
git branch -M main
git push -u origin main

9. 공개 프로젝트를 fork하여 개선하기

다른 사람의 저장소를 자신의 계정으로 복사하고 싶다면 GitHub 페이지 오른쪽 위의 Fork 버튼을 누른다.

원본 저장소
→ Fork
→ 내 GitHub 계정에 복사본 생성
→ 내 복사본을 clone
→ 수정 후 push

수업 코드를 자신의 방식으로 개선하거나 포트폴리오용 프로젝트로 발전시킬 때 적합하다.

10. 처음에는 이 명령만 기억하기

명령 의미
git clone 저장소주소 저장소를 처음 내려받기
git pull 원격 저장소의 최신 변경 사항 받기
git status 현재 변경 상태 확인
git add . 변경 파일을 commit 대상으로 추가
git commit -m "메시지" 변경 내용을 하나의 작업 단위로 기록
git push commit을 GitHub에 업로드

실습할 때 주의할 점

  1. 비밀번호, API 키, 인증 토큰은 GitHub에 올리지 않는다.
  2. 학습 데이터 전체와 대용량 모델 파일은 용량을 확인한 뒤 업로드한다.
  3. 수정 전에 git pull, 수정 후에는 git status를 먼저 확인한다.
  4. GitHub 노트북 미리보기 오류와 파일 다운로드 실패를 구분한다.
  5. 팀 프로젝트에서는 무엇을 수정했는지 알 수 있도록 commit 메시지를 구체적으로 작성한다.

마무리

학생에게 GitHub의 가장 큰 장점은 단순한 파일 보관이 아니다.

개인 노트북에서 작성한 코드를 GitHub에 올리고, GPU가 있는 클라우드 환경에서 즉시 내려받아 실행하고, 결과를 다시 공유할 수 있다는 점이 핵심이다.
