1. ResNet이 등장한 이유

CNN은 층을 깊게 쌓을수록 더 복잡한 특징을 배울 수 있다. 하지만 층이 너무 깊어지면 학습이 어려워진다. 대표적인 문제가 gradient vanishing과 degradation 문제다.

ResNet은 이 문제를 해결하기 위해 residual connection, 즉 skip connection을 사용한다. 입력을 몇 개의 layer 뒤 출력에 더해 주어, 모델이 전체 변환을 새로 배우는 대신 “입력에서 얼마나 수정할지”를 배우게 한다.

일반 CNN:
x -> Conv -> Conv -> y

ResNet block:
x -> Conv -> Conv -> + -> y
|__________________|

이 구조 덕분에 ResNet은 깊은 네트워크를 비교적 안정적으로 학습할 수 있다.

2. ResNet50의 의미

ResNet50은 residual block을 사용해 50개 층 수준의 깊이를 가진 CNN 모델이다. ImageNet으로 미리 학습된 가중치를 가져오면, 선, 모서리, 질감, 물체 일부 같은 일반 이미지 특징을 이미 알고 있는 상태에서 RPS 데이터에 맞게 미세조정할 수 있다.

base_model = keras.applications.ResNet50(
    weights="imagenet",
    include_top=False,
    input_tensor=x
)

include_top=False는 ImageNet 1000개 클래스 분류기를 제거하고, 가위/바위/보 3개 클래스용 새 분류기를 붙이기 위한 설정이다.

3. ResNet 전처리

ResNet50은 ImageNet 학습 당시 사용한 전처리 방식이 있다. 그래서 입력 이미지를 모델에 넣기 전에 ResNet 전용 preprocess_input을 적용한다.

inputs = keras.Input(shape=(IMG_SIZE, IMG_SIZE, 3))
x = keras.applications.resnet.preprocess_input(inputs)

이 때문에 데이터 로딩 직후 train_data / 255.0처럼 직접 정규화하지 않는다. 전처리는 모델 그래프 안에서 처리된다.

6. QAT는 ResNet에서도 같은 역할을 한다

ResNet50 버전에서도 QAT의 목적은 같다. TFLite 양자화 모델로 변환했을 때 정확도 손실을 줄이기 위해, 학습 중 양자화 오차를 미리 경험하게 한다.

def apply_quantization_to_layer(layer):
    if isinstance(layer, (keras.layers.Dense, keras.layers.Conv2D)):
        return tfmot.quantization.keras.quantize_annotate_layer(layer)
    return layer

annotated_model = keras.models.clone_model(
    model,
    clone_function=apply_quantization_to_layer,
)

qat_model = tfmot.quantization.keras.quantize_apply(annotated_model)

초보자 관점에서는 이 코드를 “ResNet50 모델의 주요 계산 layer에 양자화 훈련 준비 표시를 붙이고, 실제 QAT 모델로 변환하는 단계”로 이해하면 된다.

7. BatchNormalization을 고정하는 이유

전이학습에서 BatchNormalization layer는 주의가 필요하다. 이미 ImageNet에서 학습된 통계를 갖고 있는데, 작은 RPS 데이터셋으로 다시 흔들리면 성능이 불안정해질 수 있다.

for layer in base_model.layers:
    if isinstance(layer, keras.layers.BatchNormalization):
        layer.trainable = False

그래서 BatchNormalization은 고정하고, 나머지 layer를 RPS 데이터에 맞게 조정하는 방식을 사용한다.

핵심 정리

ResNet50은 residual connection으로 깊은 CNN을 안정적으로 학습하게 만든 대표 모델이다. 온디바이스 AI에서는 ResNet50이 항상 더 좋은 선택은 아니며, DenseNet121과 정확도, 모델 크기, 추론 속도를 함께 비교해야 한다.
