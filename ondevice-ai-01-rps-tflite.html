오늘 배울 것

이번 시간에는 라즈베리파이에서 카메라 영상을 받아 가위, 바위, 보를 실시간으로 분류한다.

핵심 흐름은 다음과 같다.

카메라 프레임 1장
→ 64 x 64 이미지로 줄이기
→ TFLite 모델에 넣기
→ scissors / rock / paper 중 하나 출력
→ 화면에 결과 표시

왜 TFLite를 쓰는가?

일반 TensorFlow 모델은 학습과 연구에는 좋지만 라즈베리파이 같은 작은 장치에는 무거울 수 있다.

TFLite는 작은 장치에서 실행하기 쉽게 만든 형식이다.

구분 일반 딥러닝 모델 TFLite 모델
목적 학습, 실험 추론, 배포
실행 환경 PC, GPU 서버 라즈베리파이, 스마트폰, 엣지 장치
장점 유연함 가볍고 빠름

여기서 중요한 차이는 학습용 모델과 배포용 모델의 역할이다.

학습용 모델은 여러 실험을 하기 좋게 만들어져 있다. 레이어를 바꾸고, 손실 함수를 바꾸고, 중간 결과를 확인하기 쉽다. 반면 라즈베리파이에서 필요한 것은 이미 학습이 끝난 모델을 빠르게 실행하는 것이다.

학습 단계: 정확도를 높이는 것이 중요하다.
배포 단계: 작게, 빠르게, 안정적으로 실행하는 것이 중요하다.

TFLite는 두 번째 목적에 맞춘 형식이다. 그래서 수업에서는 “TFLite는 학습을 위한 도구가 아니라 장치에서 추론을 실행하기 위한 도구”라고 설명하면 이해가 쉽다.

모델 입력과 출력 이해하기

가위바위보 모델은 이미지를 입력으로 받고 숫자 3개를 출력한다.

먼저 모델이 기대하는 입력 모양을 확인해야 한다.

print(input_details[0]["shape"])
print(input_details[0]["dtype"])

예를 들어 다음처럼 나온다면,

[ 1 64 64 3]
uint8

의미는 다음과 같다.

값 의미
1 한 번에 이미지 1장을 넣는다
64 세로 픽셀 수
64 가로 픽셀 수
3 RGB 또는 BGR 색상 채널
uint8 0부터 255 사이의 정수 이미지

입력 shape을 확인하지 않고 코드를 작성하면 가장 흔한 오류가 발생한다.

ValueError: Cannot set tensor: Dimension mismatch

이 오류는 모델이 틀렸다는 뜻이 아니라, 모델이 기대하는 입력 크기와 코드에서 넣은 이미지 크기가 다르다는 뜻이다.

예를 들어 출력이 다음과 같다고 하자.

[0.05, 0.90, 0.05]

이 값은 세 클래스에 대한 점수다.

0번: scissors
1번: rock
2번: paper

가장 큰 값은 1번이므로 결과는 rock이다.

데이터 개수를 코드로 확인하는 방법은 다음과 같다.

from pathlib import Path

DATASET_DIR = Path("RPS_Dataset")
LABELS = {
    0: "scissors",
    1: "rock",
    2: "paper",
}

total = 0
for label_id, label_name in LABELS.items():
    image_paths = list((DATASET_DIR / str(label_id)).glob("*.*"))
    count = len(image_paths)
    total += count
    print(label_id, label_name, count)

print("total:", total)

수업에서는 이 단계가 중요하다. 모델을 만들기 전에 데이터가 실제로 있는지, 클래스별 개수가 균형적인지 먼저 확인해야 한다.

이미지 파일을 배열로 읽기

기존 실습 코드의 핵심은 OpenCV로 이미지를 읽고, RGB로 바꾸고, 64 x 64로 줄이는 것이다.

import glob
import cv2
import numpy as np
from sklearn.model_selection import train_test_split

files_path = "RPS_Dataset"
IMG_SIZE = 64

train_data = []
train_labels = []
test_data = []
test_labels = []

for label_id in range(3):
    pattern = f"{files_path}/{label_id}/*.*"
    files = glob.glob(pattern)
    print(pattern, len(files))

    images = []
    labels = []

    for file_path in files:
        image = cv2.imread(file_path, cv2.IMREAD_COLOR)
        if image is None:
            print("skip broken image:", file_path)
            continue

        image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)
        image = cv2.resize(image, (IMG_SIZE, IMG_SIZE))

        images.append(image)
        labels.append(label_id)

    images = np.array(images)
    labels = np.array(labels)

    xtrain, xtest, ytrain, ytest = train_test_split(
        images,
        labels,
        test_size=0.2,
        random_state=123,
        shuffle=True,
    )

    train_data.append(xtrain)
    train_labels.append(ytrain)
    test_data.append(xtest)
    test_labels.append(ytest)

train_data = np.concatenate(train_data)
train_labels = np.concatenate(train_labels)
test_data = np.concatenate(test_data)
test_labels = np.concatenate(test_labels)

print("train_data:", train_data.shape)
print("train_labels:", train_labels.shape)
print("test_data:", test_data.shape)
print("test_labels:", test_labels.shape)

출력 shape은 보통 다음과 비슷하다.

train_data: (약 2170, 64, 64, 3)
train_labels: (약 2170,)
test_data: (약 547, 64, 64, 3)
test_labels: (약 547,)

여기서 64, 64, 3은 모델에 들어갈 이미지 한 장의 모양이다.

64: 세로 픽셀
64: 가로 픽셀
3: RGB 색상 채널

BGR을 RGB로 바꾸는 이유

OpenCV는 이미지를 기본적으로 BGR 순서로 읽는다.

OpenCV 읽기 결과: B, G, R
일반 딥러닝 모델 기대: R, G, B

그래서 다음 줄이 필요하다.

image = cv2.cvtColor(image, cv2.COLOR_BGR2RGB)

이 줄을 빼도 코드는 실행된다. 하지만 빨간색과 파란색 채널이 뒤바뀐 이미지가 모델에 들어가므로 학습과 추론 결과가 나빠질 수 있다.

기본 코드 구조

import cv2
import numpy as np
import tflite_runtime.interpreter as tflite

model_path = "RPS_PreTrained_DenseNet.tflite"
interpreter = tflite.Interpreter(model_path=model_path)
interpreter.allocate_tensors()

input_details = interpreter.get_input_details()
output_details = interpreter.get_output_details()
input_dtype = input_details[0]["dtype"]

labels = {0: "scissors", 1: "rock", 2: "paper"}

이 부분은 모델을 메모리에 올리는 과정이다. 수업에서는 이것을 AI 엔진을 켜는 단계라고 설명하면 쉽다.

모델이 실제로 어떤 입력과 출력을 기대하는지 먼저 출력해 보면 디버깅이 쉬워진다.

print("Input details")
for item in input_details:
    print(item)

print("Output details")
for item in output_details:
    print(item)

예상 출력은 모델마다 다르지만 보통 다음 정보를 확인한다.

shape: [1, 64, 64, 3]
dtype: uint8 또는 float32
index: 입력 텐서 번호

수업에서는 학생에게 “모델 설명서를 코드로 읽는 단계”라고 설명하면 좋다.

한 장 이미지 분류하기

def predict(frame):
    image = cv2.resize(frame, (64, 64))
    image = np.expand_dims(image, 0).astype(input_dtype)

    interpreter.set_tensor(input_details[0]["index"], image)
    interpreter.invoke()

    scores = interpreter.get_tensor(output_details[0]["index"])[0]
    index = int(np.argmax(scores))
    return labels[index], float(scores[index])

여기서 가장 중요한 줄은 두 개다.

interpreter.invoke()
np.argmax(scores)

interpreter.invoke()는 모델을 실행하는 명령이고, np.argmax(scores)는 가장 높은 점수를 가진 클래스를 고르는 명령이다.

코드를 조금 더 안전하게 만들려면 전처리 과정을 함수로 분리한다.

def preprocess(frame):
    image = cv2.resize(frame, (64, 64))
    image = np.expand_dims(image, axis=0)
    return image.astype(input_dtype)

수업에서는 이 부분을 다음처럼 설명하면 좋다.

카메라 이미지는 사람이 보기 위한 이미지다.
preprocess는 그 이미지를 모델이 읽을 수 있는 모양으로 바꾸는 번역기다.

만약 모델이 float32 입력을 기대한다면 보통 0부터 1 사이로 정규화해야 한다.

if input_dtype == np.float32:
    image = image.astype(np.float32) / 255.0
else:
    image = image.astype(input_dtype)

반대로 uint8 모델인데 0부터 1 사이의 float 값을 넣으면 결과가 이상해질 수 있다. 입력 dtype과 정규화 여부를 함께 확인해야 한다.

한 장 이미지 파일로 먼저 테스트하는 코드는 다음과 같다.

test_image = cv2.imread("test_rock.jpg")

if test_image is None:
    raise FileNotFoundError("test_rock.jpg 파일을 찾을 수 없습니다.")

label, confidence = predict(test_image)
print("prediction:", label)
print("confidence:", confidence)

실시간 카메라로 바로 넘어가기 전에 이미지 한 장으로 확인하면 문제를 더 빨리 찾을 수 있다.

한 장 이미지도 안 된다 → 모델 입력/출력 또는 파일 경로 문제
한 장 이미지는 된다 → 카메라, 실시간 처리, 후처리 문제

전체 실행 예제

아래 코드는 한 파일로 저장해서 실행할 수 있는 최소 예제다.

import cv2
import numpy as np
import tflite_runtime.interpreter as tflite

MODEL_PATH = "RPS_PreTrained_DenseNet.tflite"
LABELS = {0: "scissors", 1: "rock", 2: "paper"}
CONFIDENCE_THRESHOLD = 0.75

interpreter = tflite.Interpreter(model_path=MODEL_PATH)
interpreter.allocate_tensors()

input_details = interpreter.get_input_details()
output_details = interpreter.get_output_details()
input_shape = input_details[0]["shape"]
input_dtype = input_details[0]["dtype"]

height = int(input_shape[1])
width = int(input_shape[2])


def preprocess(frame):
    image = cv2.resize(frame, (width, height))
    image = np.expand_dims(image, axis=0)

    if input_dtype == np.float32:
        image = image.astype(np.float32) / 255.0
    else:
        image = image.astype(input_dtype)

    return image


def predict(frame):
    image = preprocess(frame)
    interpreter.set_tensor(input_details[0]["index"], image)
    interpreter.invoke()

    scores = interpreter.get_tensor(output_details[0]["index"])[0]
    index = int(np.argmax(scores))
    confidence = float(scores[index])
    return LABELS[index], confidence


cap = cv2.VideoCapture(0)
if not cap.isOpened():
    raise RuntimeError("카메라를 열 수 없습니다.")

cap.set(cv2.CAP_PROP_FRAME_WIDTH, 320)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 240)

while True:
    ret, frame = cap.read()
    if not ret:
        break

    label, confidence = predict(frame)
    display_label = label if confidence >= CONFIDENCE_THRESHOLD else "uncertain"

    cv2.putText(
        frame,
        f"{display_label} {confidence:.2f}",
        (20, 45),
        cv2.FONT_HERSHEY_SIMPLEX,
        0.9,
        (0, 255, 0),
        2,
    )

    cv2.imshow("RPS TFLite", frame)

    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

cap.release()
cv2.destroyAllWindows()

이 전체 코드는 다음 네 덩어리로 읽으면 된다.

코드 영역 역할
모델 로드 .tflite 파일을 메모리에 올림
preprocess() 카메라 이미지를 모델 입력 형식으로 변환
predict() 모델을 실행하고 가장 높은 점수의 클래스를 선택
카메라 루프 프레임을 계속 읽고 화면에 결과 표시

실시간 AI에서는 큰 화면보다 일정한 속도가 더 중요하다. 화면이 조금 작아도 FPS가 안정적이면 사용자 경험이 더 좋다.

실시간 인식에서 꼭 필요한 안정화

카메라 영상은 매 프레임 조금씩 흔들린다. 그래서 모델 결과도 흔들릴 수 있다.

rock → rock → paper → rock → rock

이런 경우 바로 결과를 확정하면 게임이 이상해진다. 그래서 보통 다음 규칙을 둔다.

같은 결과가 8프레임 연속 나오면 확정
confidence가 0.75 이상일 때만 인정
한 번 판정한 뒤 1.5초 쉬기

이것을 후처리라고 한다.

간단한 안정화 코드는 다음처럼 만들 수 있다.

last_label = None
stable_count = 0

def update_stable_label(label, confidence):
    global last_label, stable_count

    if confidence < 0.75:
        last_label = None
        stable_count = 0
        return "uncertain"

    if label == last_label:
        stable_count += 1
    else:
        last_label = label
        stable_count = 1

    if stable_count >= 8:
        return label
    return "waiting"

이 코드는 학생들에게 “모델 출력”과 “최종 판정”이 다르다는 점을 보여 주기에 좋다.

모델 출력: 지금 프레임 하나를 보고 낸 답
최종 판정: 여러 프레임을 보고 시스템이 확정한 답

실제 제품에서도 이 차이가 중요하다. 매 프레임의 결과를 바로 행동으로 연결하면 화면이 흔들릴 때마다 시스템도 흔들린다.

JSON 이벤트로 연결하기

화면에 글자만 표시하면 다른 시스템과 연결하기 어렵다. 그래서 결과를 JSON으로 내보내면 좋다.

{
  "user": "rock",
  "computer": "scissors",
  "result": "win",
  "confidence": 0.91
}

이렇게 만들면 웹 서버, 게임 화면, LED, 스피커, 데이터 저장 시스템과 쉽게 연결할 수 있다.

Python에서는 다음처럼 만들 수 있다.

import json
import random

def judge(user, computer):
    if user == computer:
        return "draw"
    if (user, computer) in [
        ("rock", "scissors"),
        ("scissors", "paper"),
        ("paper", "rock"),
    ]:
        return "win"
    return "lose"

computer = random.choice(["scissors", "rock", "paper"])
event = {
    "user": label,
    "computer": computer,
    "result": judge(label, computer),
    "confidence": round(confidence, 3),
}

print(json.dumps(event, ensure_ascii=False))

이렇게 출력 형식을 정해 두면 나중에 웹 화면이나 서버를 붙일 때 코드를 크게 바꾸지 않아도 된다.

아래는 안정화, 승패 계산, JSON 출력을 한 번에 넣은 예제다.

import json
import random
import time

WIN_CASES = {
    ("rock", "scissors"),
    ("scissors", "paper"),
    ("paper", "rock"),
}

last_label = None
stable_count = 0
last_event_time = 0


def judge(user, computer):
    if user == computer:
        return "draw"
    if (user, computer) in WIN_CASES:
        return "win"
    return "lose"


def get_stable_label(label, confidence, required_count=8):
    global last_label, stable_count

    if confidence < 0.75:
        last_label = None
        stable_count = 0
        return None

    if label == last_label:
        stable_count += 1
    else:
        last_label = label
        stable_count = 1

    if stable_count >= required_count:
        return label
    return None


def make_event(user_label, confidence):
    computer = random.choice(["scissors", "rock", "paper"])
    return {
        "timestamp": time.time(),
        "user": user_label,
        "computer": computer,
        "result": judge(user_label, computer),
        "confidence": round(confidence, 3),
    }


# 카메라 루프 안에서 사용
stable_label = get_stable_label(label, confidence)
now = time.time()

if stable_label is not None and now - last_event_time > 1.5:
    event = make_event(stable_label, confidence)
    print(json.dumps(event, ensure_ascii=False))
    last_event_time = now

핵심은 stable_label이 None일 때는 아직 판정하지 않는다는 점이다. 모델은 매 프레임 답을 내지만, 게임은 안정된 답이 나왔을 때만 진행한다.

자주 생기는 문제와 해결

증상 가능한 원인 해결
항상 같은 클래스로 나옴 label 순서가 모델과 다름 학습 때 클래스 순서 확인
confidence가 낮음 조명, 거리, 배경 문제 손 위치를 고정하고 밝기 확보
실행은 되지만 느림 해상도나 모델이 큼 카메라 해상도와 입력 크기 축소
Dimension mismatch 입력 shape 불일치 input_details[0]["shape"] 확인
카메라 창이 안 뜸 원격 접속 또는 GUI 문제 모니터 연결 또는 웹 출력 사용

실습 과제

  1. 카메라 화면에 scissors, rock, paper가 표시되게 한다.
  2. confidence가 0.75보다 낮으면 uncertain으로 표시한다.
  3. 같은 손동작이 여러 프레임 유지될 때만 결과를 확정한다.
  4. 컴퓨터가 랜덤으로 가위바위보를 내고 승패를 계산하게 한다.
  5. 결과를 JSON으로 출력한다.

추가 과제는 다음과 같다.

  1. 손이 화면 중앙에 있을 때만 판정하도록 ROI 영역을 만든다.
  2. 최근 30프레임의 평균 FPS를 화면에 표시한다.
  3. 틀린 결과가 나왔을 때 해당 프레임을 이미지 파일로 저장한다.
