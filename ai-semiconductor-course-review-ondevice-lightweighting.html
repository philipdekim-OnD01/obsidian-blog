1. 전체 큰 그림

관련 블로그 강의자료AI 수학 기초 TFLite 실시간 RPS 전이학습 경량화

이 과목의 핵심 흐름은 다음 한 문장으로 정리할 수 있다.

AI 모델을 수학적으로 이해하고, Python으로 학습시키며, CNN과 생성모델로 이미지와 반도체 데이터를 다루고, 최종적으로 라즈베리파이 같은 제한된 장치에서 동작하도록 배포하고 경량화하는 과정.

학기 전체는 다음 순서로 이어진다.

  1. 선형대수와 PCA: 데이터를 벡터와 행렬로 표현한다.
  2. 기초 통계와 Python: 데이터를 읽고, 전처리하고, 분포와 관계를 본다.
  3. 머신러닝: 사람이 feature를 만들고 모델이 규칙을 학습한다.
  4. 딥러닝: 모델이 feature와 판단을 함께 학습한다.
  5. CNN: 이미지처럼 공간 구조가 있는 데이터를 잘 처리한다.
  6. 생성모델: 부족한 데이터나 새로운 샘플 생성을 다룬다.
  7. 반도체 공정 데이터: 이상 탐지, 수율, 불균형 데이터 문제에 AI를 적용한다.
  8. 데이터 증강과 전이학습: 적은 데이터로 실제 환경에 강한 모델을 만든다.
  9. 온디바이스 AI: 모델을 서버가 아니라 장치에서 직접 실행한다.
  10. 경량화: 제한된 메모리, 전력, 연산량 안에서 추론 성능을 유지한다.

2. 선형대수: AI 계산의 언어

관련 블로그 강의자료벡터와 내적 행렬곱과 신경망 선형변환

2.1 벡터

벡터는 여러 숫자를 한 번에 묶은 것이다.

x = [x1, x2, x3, ...]

AI에서 하나의 데이터 샘플은 보통 벡터로 표현된다.

  • 학생 데이터: [공부시간, 출석률, 과제점수]
  • 이미지 데이터: 픽셀 값을 길게 펼친 벡터
  • 반도체 공정 데이터: 센서 값, 공정 조건, wafer 특성 값의 묶음

2.2 행렬

행렬은 여러 벡터를 모아 놓은 것이다.

X = [
  [x11, x12, x13],
  [x21, x22, x23],
  [x31, x32, x33]
]

머신러닝에서는 보통 X가 입력 데이터 전체, y가 정답이다.

X: 입력 feature 행렬
y: label 또는 target
W: 가중치 행렬
b: bias

2.3 내적과 가중합

가장 기본적인 모델 계산은 내적이다.

y = w1*x1 + w2*x2 + ... + wn*xn + b

이 식은 다음 의미를 가진다.

  • x: 입력값
  • w: 각 입력이 얼마나 중요한지 나타내는 가중치
  • b: 판단 기준을 이동시키는 bias
  • y: 모델의 출력

퍼셉트론, 선형회귀, 신경망의 dense layer는 모두 이 가중합에서 출발한다.

2.4 PCA의 직관

PCA는 고차원 데이터를 중요한 방향으로 다시 표현하는 방법이다.

핵심은 다음이다.

  • 데이터가 가장 많이 퍼진 방향을 찾는다.
  • 그 방향을 새로운 축으로 삼는다.
  • 덜 중요한 축을 줄이면 차원 축소가 된다.
  • 시각화, 노이즈 감소, feature 압축에 사용할 수 있다.

반도체 공정 데이터처럼 센서가 많고 feature가 많은 경우, PCA는 전체 변동을 요약하는 데 유용하다.

3. 미분, 손실함수, 경사하강법

관련 블로그 강의자료미분과 Chain Rule Gradient와 Backpropagation Gradient Descent Cross Entropy

3.1 미분의 의미

미분은 입력이 조금 변할 때 출력이 얼마나 변하는지 보는 것이다.

기울기 = y의 변화량 / x의 변화량

AI 학습에서는 “가중치를 조금 바꾸면 손실이 얼마나 줄어드는가”를 계산하기 위해 미분을 사용한다.

3.2 손실함수

손실함수는 모델이 얼마나 틀렸는지를 숫자로 표현한다.

회귀 문제의 대표 손실:

MSE = 평균((정답 - 예측)^2)
MAE = 평균(|정답 - 예측|)
RMSE = sqrt(MSE)

분류 문제의 대표 손실:

Cross Entropy Loss

학습의 목표는 손실을 줄이는 것이다.

좋은 모델 = 손실이 낮고, 처음 보는 데이터에서도 성능이 유지되는 모델

3.3 경사하강법

경사하강법은 손실이 줄어드는 방향으로 가중치를 조금씩 바꾸는 방법이다.

w_new = w_old - learning_rate * gradient

여기서 중요한 개념은 다음이다.

  • gradient: 손실을 증가시키는 방향
  • -gradient: 손실을 줄이는 방향
  • learning_rate: 한 번에 얼마나 이동할지 결정하는 값

학습률이 너무 크면 최적점을 지나쳐 발산할 수 있고, 너무 작으면 학습이 느리다.

4. 머신러닝: 사람이 feature를 만들고 모델이 규칙을 찾는 방식

관련 블로그 강의자료통계와 전처리 AI 반도체 매출 검증 반도체 가치사슬

4.1 머신러닝 기본 구조

데이터 → 사람이 feature 추출 → 모델 학습 → 예측

머신러닝은 정형 데이터에서 강하다. 예를 들어 반도체 데이터에서 다음과 같은 feature를 만들 수 있다.

  • wafer yield
  • defect count
  • process time
  • sensor average
  • sensor drift
  • 공정 노드 대비 생산성
  • 웨이퍼당 bit 생산량

하지만 이미지, 음성, 자연어처럼 복잡한 데이터에서는 사람이 feature를 직접 설계하기 어렵다.

4.2 회귀

회귀는 연속적인 숫자를 예측하는 문제다.

y = ax + b

예:

  • 공정 조건으로 수율 예측
  • 센서 데이터로 불량률 예측
  • 학습 시간으로 시험 점수 예측

4.3 분류

분류는 class를 예측하는 문제다.

예:

  • 정상 / 불량
  • 가위 / 바위 / 보
  • 양품 / 이상 / 고위험

4.4 평가 지표

회귀에서는 여러 지표를 함께 봐야 한다.

지표 의미 해석
MSE 오차 제곱 평균 큰 오차에 민감
RMSE MSE의 제곱근 실제 단위로 오차 해석 가능
MAE 절대오차 평균 이상치 영향이 상대적으로 작음
R2 설명력 평균 예측보다 얼마나 나은지

분류에서는 다음을 본다.

지표 의미
Accuracy 전체 중 맞춘 비율
Precision 불량이라고 예측한 것 중 실제 불량 비율
Recall 실제 불량 중 모델이 잡아낸 비율
F1-score Precision과 Recall의 균형

반도체 불량 탐지처럼 불량 데이터가 매우 적은 경우에는 accuracy만 보면 안 된다. 대부분을 정상이라고 해도 accuracy가 높게 나올 수 있기 때문이다. 이때는 recall과 F1-score가 중요하다.

5. 딥러닝: feature를 모델이 스스로 학습하는 방식

관련 블로그 강의자료작은 신경망 행렬곱과 신경망 Backpropagation

5.1 딥러닝이 나온 이유

머신러닝의 한계는 사람이 feature를 만들어야 한다는 점이다.

머신러닝: 데이터 → 사람이 feature 추출 → 모델 → 예측
딥러닝: 데이터 → 모델이 feature와 판단을 함께 학습 → 예측

딥러닝의 본질은 단순히 성능이 좋은 최신 기술이 아니라, feature engineering의 많은 부분을 모델 내부 학습으로 옮긴 구조 변화다.

5.2 퍼셉트론

퍼셉트론은 하나의 인공 뉴런이다.

입력 x → 가중합 z = Wx + b → 활성화 함수 → 출력

하나의 퍼셉트론은 본질적으로 직선 하나로 데이터를 나누는 모델이다. 따라서 선형 분리 가능한 문제는 풀 수 있지만 XOR처럼 선형 분리 불가능한 문제는 풀기 어렵다.

5.3 다층 신경망

퍼셉트론을 여러 층으로 쌓으면 복잡한 비선형 경계를 만들 수 있다.

입력층 → 은닉층 → 은닉층 → 출력층

층이 깊어질수록 단순한 feature에서 복잡한 feature로 올라간다.

이미지 예:

픽셀 → 선/모서리 → 질감/패턴 → 객체 일부 → 객체 class

5.4 활성화 함수

활성화 함수는 신경망에 비선형성을 넣는다.

대표 예:

  • Sigmoid
  • Tanh
  • ReLU
  • Softmax

활성화 함수가 없으면 여러 층을 쌓아도 결국 하나의 선형 변환과 비슷해진다. 따라서 딥러닝이 복잡한 문제를 풀려면 활성화 함수가 필수다.

5.5 역전파

역전파는 출력에서 계산한 손실을 앞쪽 layer로 전달하며 각 가중치가 손실에 얼마나 영향을 줬는지 계산하는 방법이다.

forward: 입력 → 예측 → 손실 계산
backward: 손실 → gradient 계산 → 가중치 업데이트

기울기 소실은 깊은 신경망에서 앞쪽 layer로 갈수록 gradient가 너무 작아져 학습이 거의 멈추는 현상이다. XOR 문제의 직접 원인은 기울기 소실이 아니라 단일 퍼셉트론의 선형 분리 한계다.

6. CNN: 이미지 처리를 위한 핵심 구조

관련 블로그 강의자료TFLite RPS 실습 전이학습 RPS ResNet QAT 비교

6.1 왜 CNN이 필요한가

이미지는 픽셀의 위치 관계가 중요하다. 일반 dense layer는 이미지를 길게 펼쳐서 처리하기 때문에 공간 구조를 잃기 쉽다.

CNN은 작은 필터를 이미지 위에 움직이며 지역 패턴을 찾는다.

입력 이미지 → Convolution → Activation → Pooling → Classifier

6.2 Convolution

Convolution은 필터가 이미지 일부 영역과 곱셈-덧셈을 수행하면서 feature map을 만드는 과정이다. 초기 layer는 선, 모서리, 색상 변화 같은 단순한 패턴을 잡고, 뒤 layer는 더 복잡한 형태를 잡는다.

6.3 Pooling

Pooling은 feature map의 크기를 줄인다.

효과:

  • 연산량 감소
  • 위치 변화에 조금 더 강해짐
  • 중요한 feature만 남김

6.4 전이학습

전이학습은 이미 큰 데이터로 학습된 모델을 가져와 새로운 문제에 맞게 일부만 다시 학습하는 방법이다. 가위바위보 실습에서 DenseNet 같은 pretrained 모델을 활용하면 적은 데이터로도 성능을 낼 수 있다.

기존 모델의 feature extractor + 새 분류기

7. 데이터 증강: 적은 데이터로 일반화 성능 높이기

관련 블로그 강의자료전이학습과 데이터 보강 DenseNet QAT 초보자용 GCP GPU 실습

7.1 왜 필요한가

실제 환경에서는 학습 데이터와 입력 환경이 다르다.

예:

  • 손 위치가 다름
  • 카메라 각도가 다름
  • 밝기가 다름
  • 손 크기가 다름
  • 배경이 다름
  • 카메라가 흔들림

데이터 증강은 원본 이미지를 여러 방식으로 변형해 모델이 다양한 상황을 경험하게 만드는 기술이다.

7.2 대표 증강

증강 의미
Flip 좌우 반전
Rotation 회전
Shift 위치 이동
Zoom 확대/축소
Brightness 밝기 변화
Cutout 일부 영역 가림
MixUp 두 이미지를 섞어 학습
AutoAugment 증강 전략을 자동 탐색

7.3 주의할 점

데이터 증강은 학습 데이터에만 적용해야 한다. test 데이터까지 증강하거나, train/test 분리가 잘못되면 데이터 누수가 발생한다.

올바른 순서:
원본 데이터 수집 → train/test 분리 → train에만 augmentation 적용 → test는 원본 기준 평가

8. 생성모델: 없는 데이터를 만들고 분포를 학습하는 방법

관련 블로그 강의자료통계와 전처리 반도체 가치사슬 Edge AI와 Physical AI

8.1 Autoencoder

Autoencoder는 입력을 압축했다가 다시 복원하는 모델이다.

입력 x → Encoder → latent vector z → Decoder → 복원 x'

이상 탐지에서는 정상 데이터를 잘 복원하도록 학습한 뒤, 복원 오차가 큰 데이터를 이상으로 볼 수 있다.

8.2 VAE

VAE는 latent vector를 확률분포로 학습하는 생성모델이다.

장점:

  • 데이터 분포를 학습할 수 있음
  • 새로운 샘플 생성 가능
  • 결함 데이터가 부족할 때 synthetic data 생성에 활용 가능

한계:

  • 결함 데이터가 너무 적으면 다양성을 충분히 학습하기 어렵다.
  • latent space 해석이 어렵다.
  • 생성 데이터가 실제 공정 물리와 맞는지 검증이 필요하다.
  • 장기 시계열 패턴이나 공정 drift 반영이 부족할 수 있다.

8.3 GAN

GAN은 생성자와 판별자가 경쟁하며 학습한다.

Generator: 가짜 데이터를 생성
Discriminator: 진짜와 가짜를 구분

이미지 생성에서는 강력하지만 학습이 불안정할 수 있다.

8.4 Diffusion

Diffusion 모델은 데이터에 노이즈를 점점 추가한 뒤, 다시 노이즈를 제거하는 과정을 학습한다.

장점:

  • 고품질 생성 가능
  • 안정적인 학습이 가능한 편

단점:

  • 추론 비용이 큼
  • 온디바이스에서 직접 실행하려면 경량화가 필요함

9. 반도체 공정 데이터와 AI

관련 블로그 강의자료AI 반도체 매출 검증 글로벌 반도체 시장 요약 Edge AI와 Physical AI

9.1 반도체 데이터의 특징

반도체 제조 공정은 센서가 많고, 공정 단계가 길고, 불량 데이터가 적다.

주요 특징:

  • 고차원 센서 데이터
  • 시계열 패턴
  • 공정 drift
  • 장비별 편차
  • 불량 class 불균형
  • 설명 가능성 요구

9.2 이상 탐지

이상 탐지는 정상과 다른 패턴을 찾는 문제다.

방법:

  • 통계 기반 threshold
  • PCA 기반 이상 탐지
  • Autoencoder reconstruction error
  • VAE 기반 synthetic defect 생성
  • 분류 모델의 recall/F1-score 개선

9.3 불균형 데이터

반도체 불량 데이터는 정상 데이터에 비해 매우 적다. 이때 단순 accuracy는 위험하다.

해결 방법:

  • SMOTE 같은 오버샘플링
  • class weight
  • anomaly detection
  • VAE/GAN 기반 결함 데이터 생성
  • recall, precision, F1-score 중심 평가

9.4 설명 가능성

Fab 엔지니어 입장에서는 “모델이 맞췄다”보다 “왜 이상이라고 판단했는가”가 중요하다. 설명 가능성이 필요한 이유:

  • 원인 공정 변수 파악
  • 장비 유지보수 의사결정
  • 공정 조건 개선
  • 현장 신뢰 확보

10. 온디바이스 AI

관련 블로그 강의자료온디바이스 전체 지도 TFLite 실시간 RPS GCP GPU 실습

10.1 온디바이스 AI의 의미

온디바이스 AI는 서버가 아니라 장치 내부에서 AI 추론을 수행하는 방식이다.

예:

  • 라즈베리파이에서 웹캠 입력을 받아 가위/바위/보 분류
  • 스마트폰에서 이미지 인식
  • 공정 장비 근처 edge device에서 이상 탐지
  • 로봇, 카메라, 센서 장치에서 실시간 판단

10.2 왜 필요한가

이유 설명
지연시간 서버 왕복 없이 즉시 판단
네트워크 독립성 인터넷이 불안정해도 동작
개인정보 원본 데이터를 외부로 보내지 않음
비용 서버 사용량 감소
안정성 현장 장비에서 실시간 제어 가능

10.3 라즈베리파이 실습 흐름

가위바위보 인식 실습의 전체 구조:

카메라 입력 → 이미지 전처리 → TFLite 모델 로드 → 추론 → 결과 출력

필요 요소:

  • OpenCV: 카메라 입력과 이미지 전처리
  • NumPy: 배열 계산
  • TFLite runtime: .tflite 모델 추론
  • Python 3.11 가상환경: 패키지 충돌 방지
  • 모델 파일: RPS_PreTrained_DenseNet.tflite

실습에서 중요했던 환경 이슈:

  • Raspberry Pi OS의 기본 Python 3.13과 tflite-runtime 호환성 문제
  • pyenv로 Python 3.11 환경 구성
  • numpy==1.26.4로 버전 고정
  • SSH 환경에서 cv2.imshow()를 쓰려면 display 설정 필요
  • 코드와 .tflite 모델 파일 경로를 맞춰야 함

11. 온디바이스를 위한 모델 경량화

관련 블로그 강의자료Quantization과 Pruning 경량화 강의자료 세션 QAT 초보자용 코드 ResNet QAT 비교

온디바이스에서는 정확도만 보면 안 된다. 모델이 실제 장치에서 돌아가야 한다.

11.1 온디바이스 제약

제약 의미
메모리 모델 파일과 중간 feature map을 저장할 공간 제한
연산량 CPU/NPU가 처리할 수 있는 MAC 수 제한
전력 배터리 또는 발열 제한
지연시간 실시간 응답 필요
호환성 장치 런타임이 지원하는 연산만 사용 가능

11.2 양자화

양자화는 모델의 숫자 표현을 줄이는 방법이다.

FP32 → FP16 → INT8

효과:

  • 모델 크기 감소
  • 메모리 사용량 감소
  • CPU/NPU 추론 속도 향상 가능
  • 전력 사용 감소 가능

주의:

  • 정확도가 떨어질 수 있다.
  • INT8 양자화는 representative dataset이 중요하다.
  • 장치가 INT8 연산을 잘 지원해야 효과가 크다.

11.3 Pruning

Pruning은 중요하지 않은 weight나 channel을 제거하는 방법이다.

효과:

  • 파라미터 수 감소
  • 모델 크기 감소
  • 구조적 pruning을 잘 적용하면 속도도 개선 가능

주의:

  • 단순히 weight를 0으로 만드는 비구조적 pruning은 실제 장치에서 속도 개선이 작을 수 있다.
  • pruning 후에는 fine-tuning이 필요하다.

11.4 Knowledge Distillation

Distillation은 큰 모델의 지식을 작은 모델에 전달하는 방법이다.

Teacher model: 크고 정확한 모델
Student model: 작고 빠른 모델

장점:

  • 작은 모델이 단순 label보다 teacher의 soft output을 학습해 성능을 보완한다.
  • MobileNet, EfficientNet-lite 같은 작은 모델과 함께 쓰기 좋다.

11.5 경량 모델 구조

온디바이스에서는 처음부터 작은 구조를 선택하는 것도 중요하다.

대표 구조:

  • MobileNet
  • EfficientNet-Lite
  • ShuffleNet
  • SqueezeNet
  • Tiny CNN

핵심 아이디어:

  • depthwise separable convolution으로 연산량 감소
  • channel 수 축소
  • 입력 해상도 축소
  • 불필요한 dense layer 제거

11.6 TFLite 변환

TensorFlow/Keras 모델을 라즈베리파이나 모바일에서 실행하려면 보통 TFLite로 변환한다.

Keras model → TFLite converter → .tflite model → TFLite runtime

검증해야 할 것:

  • 변환 전후 정확도 비교
  • 모델 파일 크기
  • 추론 latency
  • 메모리 사용량
  • 실제 카메라 입력에서 동작 여부

12. 핵심 수식 모음

선형 모델

행렬 표현

Y = XW + b

MSE

MSE = (1/n) * Σ(y_i - y_hat_i)^2

경사하강법

w_new = w_old - learning_rate * gradient

퍼셉트론

z = Wx + b

output = floor((input - filter + 2*padding) / stride) + 1

Softmax

softmax(z_i) = exp(z_i) / Σ exp(z_j)

14. 요약

선형대수는 AI 데이터를 표현하는 언어다. 벡터는 하나의 데이터, 행렬은 여러 데이터를 담는다. 모델은 입력과 가중치의 내적, 즉 가중합으로 출발한다.

미분과 경사하강법은 모델을 학습시키는 방법이다. 손실함수가 현재 모델의 틀린 정도를 나타내면, 경사하강법은 손실을 줄이는 방향으로 가중치를 수정한다.

머신러닝은 사람이 feature를 만들고 모델이 그 feature로 예측한다. 정형 데이터와 반도체 공정 데이터에는 유용하지만, 이미지처럼 복잡한 데이터에서는 feature engineering이 병목이 된다.

딥러닝은 feature를 모델이 스스로 학습한다. 퍼셉트론은 입력에 가중치를 곱하고 bias를 더한 뒤 활성화 함수를 통과시키는 기본 단위다. 여러 층을 쌓으면 비선형 문제를 풀 수 있다.

CNN은 이미지의 공간 구조를 활용한다. convolution은 지역 패턴을 찾고, pooling은 크기를 줄이며, 뒤 layer로 갈수록 더 복잡한 feature를 학습한다.

데이터 증강은 실제 환경의 변화에 강한 모델을 만들기 위한 방법이다. 밝기, 위치, 회전, 확대, 가림 같은 변형을 학습 데이터에 적용해 일반화 성능을 높인다.

생성모델은 데이터 분포를 학습하고 새로운 샘플을 만든다. Autoencoder는 복원 오차로 이상 탐지에 쓸 수 있고, VAE는 결함 데이터가 부족한 반도체 공정에서 synthetic data 생성에 활용될 수 있다. 하지만 생성 데이터의 다양성, 물리적 타당성, 설명 가능성을 반드시 검토해야 한다.

온디바이스 AI는 모델을 서버가 아니라 장치에서 직접 실행하는 방식이다. 라즈베리파이 실습에서는 카메라 입력을 OpenCV로 처리하고, TFLite 모델로 추론해 가위바위보를 분류했다.

온디바이스에서는 정확도만으로는 부족하다. 모델 크기, 메모리, 지연시간, 전력, 런타임 호환성을 함께 봐야 한다. 이를 위해 양자화, pruning, distillation, MobileNet 계열 경량 모델, TFLite 변환을 사용한다.

한 학기 전체의 결론은 다음이다.

수학 기초 → 데이터 이해 → ML/DL 모델 학습 → CNN/생성모델 활용
→ 반도체와 이미지 실습 → TFLite 배포 → 온디바이스 경량화

15. 암기 문장

  • AI 모델의 기본 계산은 입력 × 가중치 + bias다.
  • 학습은 손실을 줄이는 방향으로 가중치를 바꾸는 과정이다.
  • 머신러닝은 feature를 사람이 만들고, 딥러닝은 feature를 모델이 배운다.
  • CNN은 이미지의 공간적 패턴을 convolution으로 학습한다.
  • 데이터 증강은 실제 환경 변화에 견디는 모델을 만들기 위한 방법이다.
  • 반도체 데이터는 고차원, 시계열, 불균형, 설명 가능성이 핵심 이슈다.
  • 온디바이스 AI는 정확도뿐 아니라 속도, 메모리, 전력, 호환성이 중요하다.
  • 경량화의 목표는 성능을 최대한 유지하면서 모델을 작고 빠르게 만드는 것이다.
